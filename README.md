# 🔗 API Practice Project

## 📌 Project Overview
This project demonstrates hands-on practice with API testing and interaction using real-world scenarios. 
It focuses on validating API endpoints, request-response handling, and ensuring data integrity through structured testing techniques.

## 🎯 Objectives
- Understand REST API architecture and workflow
- Perform API testing using industry-standard tools
- Validate API responses and data consistency
- Practice CRUD (Create, Read, Update, Delete) operations

## 🛠️ Tools & Technologies
- Postman (API Testing)
- REST API
- JSON
- HTTP Methods (GET, POST, PUT, DELETE)
- Git & GitHub

## 📋 API Testing Scope
- Request & Response Validation
- Status Code Verification
- Header & Body Validation
- Data Integrity Testing
- Error Handling Testing

## 🔄 HTTP Methods Covered
- GET → Retrieve data  
- POST → Create new data  
- PUT → Update existing data  
- DELETE → Remove data  

## 🧪 Test Scenarios
- Valid API request execution  
- Invalid input handling  
- Authentication   
- Response time checking  
- Edge case testing  

## 📊 Key Features
- Tested multiple API endpoints  
- Validated JSON response structure  
- Ensured proper status codes (200, 201, 400, 404, etc.)  
- Performed positive & negative testing  

## 🐞 Defect Reporting
- Identified API issues and inconsistencies  
- Documented bugs with request/response details  
- Verified fixes after retesting  

## 🔄 Testing Process
1. Requirement Understanding  
2. API Endpoint Analysis  
3. Test Case Design  
4. Execution using Postman  
5. Bug Reporting  
6. Retesting & Validation  

## 📈 Outcome
- Improved API reliability and performance  
- Ensured accurate data exchange between systems  
- Strengthened API testing skills
